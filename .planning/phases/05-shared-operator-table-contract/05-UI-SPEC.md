## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps, compact review-action button spacing |
| sm | 8px | Table cell horizontal padding, identity-panel fields, compact control gaps |
| md | 16px | Default grouped-control spacing and section rhythm |
| lg | 24px | Major surface spacing when a table section needs separation |
| xl | 32px | Reserved for wide layout gaps only |
| 2xl | 48px | Reserved for page-level breaks; do not introduce in Phase 5 |
| 3xl | 64px | Reserved for page-level breaks; do not introduce in Phase 5 |

Phase 5 spacing updates must stay on this grid: table cells `4px 8px`, tabs and headers `8px 16px`,
review-action buttons `4px 8px`, and compact inline gaps `4px`. Border radius is governed by the visual
contract and must use `4px` when touched. Do not add new non-grid spacing exceptions in Phase 5. Keep
Rich table padding medium-owned.

## Visual Hierarchy

The primary visual anchor is the active operator table for the selected surface. The active surface
tab is the secondary anchor. Pause/resume controls and operator identity are tertiary controls in the
topbar. Selected review rows use the accent outline only when the Compliance surface is active; row
hover and focus treatment must not compete with status, priority, or review-state colors.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Light `#f7f8fa`; dark `#111315` | Page background and operator-console negative space |
| Secondary (30%) | Light `#ffffff` / `#f1f3f6`; dark `#171a1d` / `#202428` | Tables, surface panels, metric strip, tabs, controls |
| Accent (10%) | `#0f766e` | Selected review row outline, subtle table-row hover, focus outline when a selected table row needs emphasis |
| Destructive | `#b42318` | Failed status, rejected review state, destructive review action border/status only |

Accent reserved for: selected review row outline, subtle table-row hover background, and focused selected-row
emphasis. Do not use accent for all buttons, labels, IDs, or generic table headers.

Existing semantic status colors remain data-state colors only: ok `#16703c`, warning `#a15c07`, info `#0969da`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Pause workers` is the primary control label; paired control remains `Resume workers` |
| Empty state heading | none - empty states appear as a single table row inside the affected table |
| Empty state body | Tasks: `No tasks in queue.`; Workers: `No workers connected yet.`; Review gaps: `No human review or verification gaps.`; Events: `No events recorded.` |
| Error state | WUI: `Control plane unavailable` plus the error detail and `Retry`; TUI: caption prefix `error: {message}` |
| Destructive confirmation | `Reject selected review gap`: prompt `Review comment`; `Request changes`: prompt `Requested changes`; cancel aborts. Phase 5 adds no new destructive actions. |

Compact TUI empty-state aliases are allowed: `(no tasks)`, `(no workers)`, `(no gaps)`, `(no events)`.
WUI null cells use an em dash; TUI null cells use `-` for terminal compatibility.

## Shared Operator Table Contract

Source of truth: implement the contract as pure metadata in `whilly/operator_views.py` or a nearby pure module.
Both `whilly/cli/tui.py` and the WUI Jinja templates must consume the same field keys and canonical labels unless
this spec marks a medium-specific label.

### Surface Order

| Order | Surface | Shared label |
|-------|---------|--------------|
| 1 | `overview` | Overview |
| 2 | `compliance` | Compliance |
| 3 | `plans_tasks` | Plans/Tasks |
| 4 | `workers` | Workers |
| 5 | `events` | Events |

Hotkeys and tab order remain `1-5`. This phase must not change pause/resume, refresh, filter, quit, or review
decision semantics.

### Task Table

Canonical row model: `OperatorTaskRow`.

| Order | Field key | Canonical label | WUI | TUI | Value contract |
|-------|-----------|-----------------|-----|-----|----------------|
| 1 | `task_id` | Task | show | show | Task ID as code/monospace |
| 2 | `plan_id` | Plan | show | show | Plan ID |
| 3 | `status` | Status | show | show | Raw status text with status color |
| 4 | `priority` | Priority | show | show | Raw priority text with priority color |
| 5 | `claimed_by` | Worker | show | show | Worker/claim owner; null as WUI em dash, TUI `-` |
| 6 | `human_review` | Review | show | show | `pending`, `approved`, `rejected`, `changes requested`, with stage in parentheses or inline code when present |
| 7 | `updated_at` | Updated | show | optional omission | WUI shows human time with ISO datetime; TUI may omit for width, but if shown the label is `Updated` |

Required Phase 5 alignment: replace WUI task label `Claimed by` with canonical `Worker`, or document it in code
as a deliberate WUI alias with a contract test. Prefer the canonical visible label `Worker`.

### Worker Table

Canonical row model: `WorkerRow`.

| Order | Field key | Canonical label | WUI | TUI | Value contract |
|-------|-----------|-----------------|-----|-----|----------------|
| 1 | `worker_id` | Worker | show | show | Worker ID as code/monospace |
| 2 | `hostname` | Hostname | show | show | Hostname; TUI compact alias `Host` is allowed only as a medium-specific label |
| 3 | `owner_email` | Owner | show | show | Owner email; null as WUI em dash, TUI `-` |
| 4 | `status` | Status | show | show | Raw status text with online/offline color |
| 5 | `last_heartbeat` | Last heartbeat | show | show | WUI shows human time with ISO datetime; TUI may use `HH:MM:SS`; TUI compact alias `Heartbeat` is allowed |

Required Phase 5 alignment: order both surfaces by field key as `worker_id`, `hostname`, `owner_email`,
`status`, `last_heartbeat`. WUI currently shows Owner before Hostname; align the order or document the difference
with a contract test.

### Review Queue Table

Canonical row model: `ReviewGap`.

| Order | Field key | Canonical label | WUI | TUI | Value contract |
|-------|-----------|-----------------|-----|-----|----------------|
| 0 | `selected` | Sel | not as column | show | TUI-only selection marker `>`; WUI uses selected row outline and `aria-selected` |
| 1 | `task_id` | Task | show | show | Task ID as code/monospace where possible |
| 2 | `plan_id` | Plan | show | show | Plan ID |
| 3 | `reason` | Reason | show | show | Review or verification gap reason |
| 4 | `stage_id` | Stage | show | show | Stage ID; null as WUI em dash, TUI `-` |
| 5 | `reviewer` | Reviewer | show | show | Reviewer; null as WUI em dash, TUI `-` |
| 6 | `actions` | Actions | show | show | `approved`, `rejected`, `changes_requested` mapped to WUI `A/X/C` buttons and TUI `a/x/c` hotkeys |

Action labels remain compact in Phase 5. Clearer labels, tooltips, confirmations, or undo affordances are Phase 7.
Only actionable `ReviewGap` rows may offer write controls; inferred/read-only gaps show `read-only` in WUI and `-`
in TUI.

### Event Table

Canonical row model: `EventRow`.

| Order | Field key | Canonical label | WUI | TUI | Value contract |
|-------|-----------|-----------------|-----|-----|----------------|
| 1 | `event_id` | Id | show | show | Event ID as code/monospace where possible |
| 2 | `task_id` | Task | show | show | Task ID; null as WUI em dash, TUI `-` |
| 3 | `plan_id` | Plan | show | show | Plan ID; null as WUI em dash, TUI `-` |
| 4 | `event_type` | Type | show | show | Event type string |
| 5 | `created_at` | At | show | show | WUI shows human time with ISO datetime; TUI may use `HH:MM:SS` |

Events are already aligned. Phase 5 should pin this contract with tests, not redesign the event table.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Filter | The global filter continues to hide rows by `data-filter-text` in WUI and `filter_snapshot()` in TUI. Field coverage must stay equivalent for tasks, workers, review gaps, and events. |
| Selection | Review selection applies only on the Compliance surface and only to actionable review rows. TUI uses `Sel`; WUI uses `aria-selected` plus accent outline. |
| Refresh | WUI continues HTMX/SSE refresh; TUI continues polling/manual `r`. Table-contract work must not freeze WUI updates while workers are paused. |
| Review actions | `a`/`x`/`c` remain Compliance-only and map to `approved`, `rejected`, `changes_requested` through the shared review-decision path. |
| Medium differences | Only documented differences are allowed: WUI can show `Updated`; TUI can omit it. TUI can use compact labels `Host` and `Heartbeat`; WUI should use canonical full labels. |
